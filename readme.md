# ML Fault Detection Challenge  

## 🧾Project Description

This project presents a Machine Learning solution for a **Binary Fault Detection System** developed as part of the IEEE Student Branch GEHU ML Challenge.

The objective is to classify whether a device is operating under normal conditions or experiencing a fault, based on 47 numerical input features generated by an embedded monitoring system.

### 🎯 Problem Statement

Given:

- **47 numerical features (F01–F47)**
- Automatically generated system measurements

Predict:

- **Class = 0 → Normal**
- **Class = 1 → Faulty**

---

## 📊 Dataset Description

### Training Data (`TRAIN.csv`)
- Features: F01–F47  
- Target: `Class`

### Test Data (`TEST.csv`)
- Features: F01–F47  
- `ID` column included

---

## Model Overview

The model was trained using random forest classifier on the provided training dataset.

Key steps:
- Data preprocessing
- Model training
- Model evaluation
- Find best Model using Hyperparameter tuning
- Model serialization for inference

The final trained model is stored inside:
    /models/best_model.pkl
